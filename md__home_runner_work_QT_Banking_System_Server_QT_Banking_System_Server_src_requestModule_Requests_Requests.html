Banking_System_Server  1.0.0
Qt-based banking app for user/admin account management, transactions, secure server communication via PostgreSQL/Supabase.
Requests Documentation for Bank System

This document describes the requests that can be sent to the server and the responses that can be received from the server.

1. Login Request

Send Request:

{
    "Request": 1,
    "Data":
        {
        "email": "user@example.com",
        "password": "pass"
        }
}

Receive Response:

On Success:

{
    "Response": 1,
    "Data": {
        "status": 1,
        "first_name": "User",
        "role": "user",
        "message": "Login successful"
    }
}

On Failure:

{
    "Response": 1,
    "Data": {
        "status": 0
        "message": "email not found"
    }
}

Or

{
    "Response": 1,
    "Data": {
        "status": 0
        "message": "Invalid password"
    }
}

2. Get Account Number (for Logged-in user)

Send Request:

{
    "Request": 2,
    "Data": {

    }
}

On Failure:

{
    "Response": 2,

        }
}

or

{
    "Response": 2,
    "Data": {
 "status": 0,
        "message": "email not found"
        }
}

3. Get Balance

Send Request:

{
    "Request": 3,
    "Data": {
        "account_number": 123456789
    }
}

Receive Response:

On Success:

{
    "Response": 3,
    "Data": {
    "status": 1,
"message": "Balance fetched successfully",
        "balance": 500.0
       
    }
}

On Failure: (will not Usually happen)

{
    "Response": 3,
    "Data": {
 "status": 0,
        "message": "Account not found"
    }
}

5. Make Transaction

Send Request:

Using email:

{
    "Request": 5,
    "Data": {
"from_account_number": 123456789,
        "to_account_number": -1,
"to_email": "user@mail.com",
        "transaction_amount": 100.0
    }
}

Using account number:

{
    "Request": 5,
    "Data": {
"from_account_number": 123456789,
        "to_account_number": 1222255555,
"to_email": "",
        "transaction_amount": 100.0
    }
}

Receive Response:

On Success:

{
    "Response": 5,
    "Data": {
        "status": 1,
        "message": "Transaction successful"
    }
}

On Failure:

{
    "Response": 5,
    "Data": {
        "status": 0,
        "message": "Insufficient balance"
    }
}

7. Get Database (for admin)

Send Request:

{
    "Request": 7,
    "Data": {
        "email": "admin@mail.com",
    }
}

Receive Response:

On Success:

{
    "Response": 7,
    "Data": {
"status": 1,
"message": "Database fetched successfully",
        "users": [
            {
                "account_number": 123456789, // NULL if admin
                "first_name": "Admin",
                "last_name": "User",
                "email": "email@mail.com",
                "role": "user",
                "balance": 500.0
            },
            {
                "account_number": NULL, // NULL if admin
                "first_name": "User",
                "last_name": "User",
                "email": "admin@mail.com",
                "role": "admin",
                "balance": 500.0
            }
        ]
    }
}

On Failure:

{
    "Response": 7,
    "Data": {
"status": 0,
         "message": "Cannot get database. User is not an admin",
    }
}

or

{
    "Response": 7,
    "Data": {
"status": 0,
         "message": "No data found",
    }
}

8. Create New User (for admin)

Send Request:

{
    "Request": 8,
    "Data": {
        "email": "admin@mail.com",
        "newUser": [
            {
                "first_name": "New",
                "last_name": "User",
                "email": "newuser@example.com",
                "password": "newpass",
                "role": "user",
                "initial_balance": 500.0
            }
        ],
    }
}

Receive Response:

On Success:

{
    "Response": 8,
    "Data": {
        "status": 1,
        "message": "New user created successfully"
    }
}

or

{
    "Response": 8,
    "Data": {
        "status": 1,
        "message": "New admin created successfully"
    }
}

On Failure:

{
    "Response": 8,
    "Data": {
        "status": 0,
        "message": "User already exists"
    }
}

or

{
    "Response": 8,
    "Data": {
        "status": 0,
        "message": "Cannot create new user. User is not an admin"
    }
}

or

{
    "Response": 8,
    "Data": {
        "status": 0,
        "message": "Missing required fields"
    }
}

or

{
    "Response": 8,
    "Data": {
        "status": 0,
        "message": "Invalid role"
    }
}

or

{
    "Response": 8,
    "Data": {
        "status": 0,
        "message": "Admin can't have account and initial balance"
    }
}
